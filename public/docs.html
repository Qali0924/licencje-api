Integracja API

Zabezpiecz swój projekt w kilka minut korzystając z globalnej infrastruktury Synthesized Studio.

1

Wprowadzenie

System walidacji opiera się na żądaniach HTTP. Twój plugin wysyła klucz licencyjny do naszego API, które weryfikuje jego poprawność w czasie rzeczywistym. W przypadku braku aktywnej licencji, zalecamy natychmiastowe zatrzymanie logiki pluginu, aby zabezpieczyć swoją pracę.
2

Endpoint Walidacji

Metoda HTTP

POST

Endpoint URL

https://licencje-api-u0sq.onrender.com/api/validate

Nagłówek (Headers)

Content-Type: application/json

arrow_forward Ciało Żądania (Request) 

{
  "key": "TWOJ-KLUCZ-LICENCYJNY",
  "ip": "AUTO"
}

Pole ip z wartością "AUTO" oznacza, że nasze API automatycznie wykryje adres IP serwera, z którego wysyłane jest żądanie. Możesz również podać konkretny adres IP, jeśli jest to wymagane.

arrow_back Ciało Odpowiedzi (Response)

Sukces (Ważna licencja)

{
  "valid": true
}

Błąd (np. Zły Klucz)

{
  "valid": false,
  "reason": "Invalid license key."
}

code Główna klasa walidacji (LicenseManager.java)

Java 17+ Code 
import com.google.gson.Gson;
import org.bukkit.Bukkit;
import org.bukkit.plugin.java.JavaPlugin;

import java.net.URI;
import java.net.http.HttpClient;
import java.net.http.HttpRequest;
import java.net.http.HttpResponse;
import java.time.Duration;
import java.util.Map;

public class LicenseManager {
    private static final String API_URL = "https://licencje-api-u0sq.onrender.com/api/validate";
    private static final Gson GSON = new Gson();
    private static final HttpClient HTTP_CLIENT = HttpClient.newBuilder()
            .version(HttpClient.Version.HTTP_2)
            .connectTimeout(Duration.ofSeconds(10))
            .build();

    public static boolean validate(JavaPlugin plugin, String licenseKey) {
        // 1. Sprawdź, czy klucz nie jest pusty
        if (licenseKey == null || licenseKey.trim().isEmpty() || licenseKey.equals("TWÓJ-UNIKALNY-KLUCZ")) {
            plugin.getLogger().severe("[Synthesized] Klucz licencyjny nie został ustawiony w config.yml!");
            return false;
        }
        
        plugin.getLogger().info("[Synthesized] Trwa weryfikacja licencji...");
        
        // 2. Przygotuj ciało żądania w formacie JSON
        String requestBody = GSON.toJson(Map.of("key", licenseKey, "ip", "AUTO"));

        try {
            // 3. Zbuduj i wyślij żądanie POST
            HttpRequest request = HttpRequest.newBuilder()
                    .uri(URI.create(API_URL))
                    .header("Content-Type", "application/json")
                    .POST(HttpRequest.BodyPublishers.ofString(requestBody))
                    .build();

            HttpResponse<String> response = HTTP_CLIENT.send(request, HttpResponse.BodyHandlers.ofString());

            // 4. Sparsuj odpowiedź JSON
            LicenseResponse licenseResponse = GSON.fromJson(response.body(), LicenseResponse.class);
            
            if (licenseResponse != null && licenseResponse.valid()) {
                plugin.getLogger().info("[Synthesized] Licencja została pomyślnie zweryfikowana. Dziękujemy!");
                return true;
            } else {
                String reason = licenseResponse != null ? licenseResponse.reason() : "Nieznany błąd walidacji";
                plugin.getLogger().severe("[Synthesized] Weryfikacja licencji nie powiodła się: " + reason);
                return false;
            }

        } catch (Exception e) {
            plugin.getLogger().severe("[Synthesized] Nie można połączyć się z serwerem licencji. Sprawdź połączenie internetowe.");
            e.printStackTrace();
            return false;
        }
    }
}

power Użycie w klasie głównej pluginu (np. Main.java)

Java 17+ Code 
public class Main extends JavaPlugin {
    
    @Override
    public void onEnable() {
        // Zapisz domyślny config.yml, jeśli nie istnieje
        this.saveDefaultConfig();
        String licenseKey = this.getConfig().getString("license-key");
        
        // Wykonaj walidację w nowym wątku, aby nie blokować głównego wątku serwera
        Bukkit.getScheduler().runTaskAsynchronously(this, () -> {
            boolean isValid = LicenseManager.validate(this, licenseKey);
            
            if (!isValid) {
                // Jeśli licencja jest nieważna, wyłącz plugin
                Bukkit.getScheduler().runTask(this, () -> {
                    getLogger().severe("Plugin zostanie wyłączony z powodu problemu z licencją.");
                    Bukkit.getPluginManager().disablePlugin(this);
                });
                return;
            }

            // Tutaj umieść logikę ładowania pluginu (rejestrowanie komend, eventów itp.)
            // Wykonaj ją z powrotem w głównym wątku serwera!
            Bukkit.getScheduler().runTask(this, () -> {
                // registerCommands();
                // registerListeners();
                getLogger().info("Plugin został poprawnie załadowany.");
            });
        });
    }
}

Kody Odpowiedzi i Obsługa Błędów

Zrozumienie kodów statusu HTTP pomoże w debugowaniu problemów. Twój plugin powinien odpowiednio reagować na różne scenariusze.

  • 200 OK

    Żądanie przetworzone pomyślnie. Sprawdź pole valid w odpowiedzi JSON, aby poznać ostateczny status licencji.

  • 400 Bad Request

    Serwer nie mógł zrozumieć żądania, prawdopodobnie z powodu błędnej składni JSON. Upewnij się, że wysyłane dane są poprawne.

  • 429 Too Many Requests

    Wysłano zbyt wiele żądań w krótkim czasie. Nasze API stosuje rate limiting, aby zapobiec nadużyciom. Plugin nie powinien ponawiać próby natychmiast.

  • 5xx Server Error

    Wystąpił problem po stronie naszego serwera API. W takim przypadku zalecamy traktowanie licencji jako tymczasowo nieważnej i ewentualne ponowienie próby po pewnym czasie.
